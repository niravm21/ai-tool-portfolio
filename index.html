Stock Earnings Scraper

Monitors a dynamic watchlist of stocks, monitors for earnings releases, extracts key KPIs from results statements using AI, and delivers a formatted comparison email automatically.

How it works

1
A watchlist of stocks is maintained in Excel, with KPIs and model estimates defined per company
2
The tool monitors the RNS feed every 5 minutes from the cloud until an earnings statement is detected
3
Claude AI scans the results statement and extracts the relevant KPIs in structured form
4
Actuals are compared against internal model and consensus estimates; a formatted email is sent immediately

Current status

Proof of concept. Rather than monitoring live RNS feeds, a PDF of the latest Porsche quarterly statement was used as input. Model and consensus numbers are illustrative. Live feed integration and cloud deployment are the next steps to production.

Tools used and why

Tool Rationale
Python Core scripting language for the monitoring loop, PDF parsing, email formatting, and orchestration logic.
Claude AI (Anthropic API) Reads the earnings PDF and extracts the relevant KPIs in structured form — replacing brittle rule-based parsing with flexible AI extraction.
Excel Stores the stock watchlist, KPI definitions, internal model estimates, and consensus figures in a format easy to update without touching code.
Claude Code The tool was built entirely using Claude Code — used to design the architecture, write all Python scripts, iterate on features, and produce the final output.
GitHub Version control and deployment source for running the tool in a cloud environment.
SMTP / Gmail Sends the formatted HTML earnings email directly to the recipient with no third-party email service required.

Daily News Aggregator

Aggregates 20+ daily news emails from a dedicated inbox, compiles the most important stories ranked by frequency of coverage across sources, and delivers a structured morning briefing automatically.

How it works

1
A dedicated inbox receives 20+ news emails per day from multiple financial and market sources
2
Claude reads and aggregates all incoming emails, extracting the key stories from each
3
Stories are ranked by importance, with the most widely covered leading
4
A structured briefing email is compiled and sent automatically each morning

Tools used and why

Tool Rationale
Claude (Anthropic) Reads and synthesises all incoming emails, identifies the most covered stories, and writes the structured briefing in a consistent format.
Claude Scheduled Tasks Runs the workflow automatically each morning without any manual trigger — no server or cron job required.
Gmail Dedicated inbox acts as the aggregation point for all incoming news sources; Claude reads directly from it via API integration.
Python Handles email retrieval, processing logic, and output formatting.

CV Screener — Weighted Skills Scoring

Extracts key skills from a job description and scores uploaded CVs against them, ranking candidates automatically.

How it works

1
Upload job description
2
Claude identifies required skills and assigns suggested weightings to each
3
Add contextual criteria (seniority, experience, etc) and accept or change the suggested weighting for each skill
4
Tool screens the CVs of the full applicant pool and ranks all candidates by weighted average skills score
5
Top 100 returned for manual review until a sufficient interview shortlist is identified

AI Operations Agent — Project Tracking Tool

A browser-based ops dashboard for tracking team commitments, monitoring deadlines, receiving automated alerts, and generating plain-English weekly summaries. No technical setup required.

Live tool

View the tool →

How it works

1
Team commitments and deadlines are logged directly in the dashboard, taking under 2 minutes per session
2
Items with no update in 3 or more days are flagged automatically; a red alert panel appears 3 days before any deadline
3
A deadline alert email fires automatically to the team, and a plain-English weekly summary email sends every Friday on a set schedule or on demand

Tools used and why

Tool Rationale
Claude (Anthropic) The prototype was built entirely using Claude — used to design the architecture, write all HTML/CSS/JavaScript, iterate on features, and produce the project brief.
HTML / JS / CSS Single file, no framework, no build step. Opens in any browser — zero barrier for non-technical users.
Netlify + Notion Netlify hosts the file in one minute; Notion embeds it so the team never leaves their existing workspace.
EmailJS Sends two automated emails from the browser — a scheduled weekly summary and a triggered 3-day deadline alert — with no backend needed.

Built with

HTML / JS / CSS Claude (Anthropic) EmailJS Netlify Notion
